---
title: "Field notes: serving on gfx906 (AMD MI50/MI60)"
---

Practical findings from serving and fine-tuning on retired-datacenter AMD **gfx906** GPUs (Instinct MI50/MI60, 32 GB Vega20) with llama.cpp under ROCm. Hardware-specific, but each pitfall generalizes — the terms used here are defined in the rest of the glossary.

## Benchmark quants before assuming smaller = faster

On gfx906, a **Q8_0 split across two GPUs decodes faster than IQ4_XS on one** (~100 vs ~80 tok/s). The IQ dequant kernels are expensive on this architecture while Q8_0's are nearly free. Quant choice is a *per-architecture* speed question, not just a size question — benchmark on your actual hardware.

## mmap thrash looks like a hang

Loading a GGUF **larger than system RAM** with the default mmap loading looks like the server froze: the page cache thrashes and re-reads the file forever. `--no-mmap` loads the same model in under a minute. If a big-model load "hangs", check RAM vs file size before blaming the GPU.

## Benchmark drift: only trust interleaved A/B

Restart-to-restart throughput on this stack varies **±20%**. Comparing "before" from yesterday's server against "after" from today's measures noise. Only compare runs interleaved on a *single* server instance.

## MTP heads are tunable — straight from the GGUF

The **MTP / nextn** draft head of a model can be fine-tuned without the original safetensors checkpoint: extract the head tensors from the served GGUF, capture the head's runtime inputs (token embeddings + final hidden states) by running a corpus through the full model, train the head in PyTorch, and re-quantize it back into the GGUF in place. Training on a corpus *generated by the served model itself* matches the training distribution to the serving distribution — on Ornith-1.0-35B this took draft acceptance at 32k context from 57% to 65–70% and long-context throughput from ~70 to ~78 tok/s, trained in ~2 h on one 32 GB card.

## Capture callbacks fire per ubatch

llama.cpp's eval callback fires once per **ubatch**, not once per chunk. A naive capture tool that reads its buffers per callback silently keeps only the *last* ubatch's rows for any chunk longer than `-ub` (default 512). Set `-b`/`-ub` at least as large as your chunk size, and verify captured row counts match token counts.
